# Security Policy

## Supported Versions

Please refer to [SUPPORTED_VERSIONS.md](SUPPORTED_VERSIONS.md) for information about supported releases.

## Reporting a Vulnerability

We take the security of our services seriously. If you believe you have found a security vulnerability in this repository, please do **not** report it via a public Github issue or pull request.

Instead, please report the vulnerability by emailing us or opening a draft security advisory on GitHub.

When reporting a vulnerability, please include:
- A description of the issue.
- The repository version/branch affected.
- A proof of concept (PoC) or steps to reproduce if available.

We will acknowledge receipt of your report within 48 hours and work with you to resolve the issue gracefully.
